#소프트웨어

소프트웨어저작권협회 : www.spc.or.kr/index2.asp 

SW백문백답 : www.spc.or.kr/custcenter/faq_list.asp 

인공지능과 저작권 (1)

  - 인공지능으로 만든 것은 저작권 인정 안됨

오픈소스SW (3)

  - 소스코드와 제공되는 파일확인(notice/readme/copyright) 또는 해당 사이트에서 라이선스 정책 확인

  - 한국저작권위원회 : 오픈소스SW 라이선스 종합정보시스템 www.olis.or.kr

     . 라이선스 검사 서비스 : 코드아이

  - 정보통신산업진흥원(NIPA) : www.oss.kr

     . 중소기업 대상 라이선스 검증 지원 등

  - 오픈소스 소프트웨어 자유롭게 이용 가능

  - 회사 내부적으로 사용할 경우에는 소스코드 공개 필요하지 않음 / 솔루션 사업을 한다면 심각한 문제 발생 가능

폰트 저작권 (5)

  - pdf에 이미지 형태로 저장된 폰트는 사용 가능

  - 이미지 파일에 폰트 사용한 것만으로는 저작권 위반 아님 / AI, EPS 파일 형태로 전달 받은 경우 중대한 문제 발생 가능

  - 블로그 등에서 무료폰트라고 적힌 것을 사용하더라도 저작권 내용 확인은 본인이 해야 함

  - 폰트 구매해서 CI 만드는데 활용 : 보통 별도 라이선스 존재함. (기업로고, 이러닝, e북, 영상, 게임 등)

  - 폰트는 미술저작물이 아니라 컴퓨터프로그램저작물로 보호됨

불법SW사용과 처벌근거 (4)

  - 불법 개발툴로 활용하여 만든 소프트웨어라도 저작권은 보호됨 (혹시 API. 등을 활용한 경우라면 해당 부분 제외)

  - 형사, 민사 소송 또는 한국저작권위원회에 조정 요청

  - 처벌근거 : 제136조, 

    . 제140조 원칙적으로는 친고죄, 하지만 상습적으로 행한 경우 비친고죄 적용 가능

    . 제141조 양벌규정

  - 저작권법 규정상 기업의 불법소프트웨어 사용은 친고죄로 처벌

양벌규정 (3)

  - 퇴사한 직원의 PC도 저작권 침해

  - 상당한 주의와 감독 의무 위반, 법인이 증명해야 함, 계약서/서약서 관리 정도로는 안됨

SW 개발 및 유지보수 (2)

  - 유지보수 과정에서 부득이하게 약간의 수정이 필요한 경우에는 가능  /  상기 범위를 벗어나는 변경은 2차적저작물작성권 침해

  - 퇴사직원경쟁업체취업 : 경영금지약정(재취업 어느 정도 방지 가능하나 근본적으로 막을 수 없음), 영업비밀 침해, 저작권(복제권, 2차적저작물작성권) 또는 특허권 침해 소송

공문 / 단속 / 감사 (2)

  - 감사를 거부한다고 하여 저작권 위반은 아님, 계약으로서 효력이 있을 수 있음

  - 감사의 방법과 조건, 범위, 일정, 대상, 비용 등을 꼼꼼히 확인하기 바람

  - 저작권사의 파트너사인 유통사는 원칙적으로는 고소권이 없음, 고소를 촉구하거나 고발을 할 수 있음

  - 저작권사와 협력하여 행동할 수 있으나 해당 유통사에서만 구입해야 한다는 것은 받아들일 수 없음

개인 노트북 및 SW의 기업 사용 (3)

  - 기본적으로 복제권 침해임

  - 불법복제 소프트웨어 사용하는 개인 노브북을 회사에 가져오면 업무용으로 볼 수 있음

소프트웨어 양도/양수 (2)

  - 리이선스 내용에 따라 다를 수 있음

  - 회사 인수/합병/분할 : FPP(패키지 박스 형태) 독립적 양도 가능 (라이선스 내용 확인 필요), 번들 형식은 가능 (PC에 종속된 계약이므로)

SW 라이선스 (10)

  - 저작권 불명인 저작물 이용 : 상당한 노력 기울인 이후 공탁 가능 (한국저작권위원회)

  - 프리웨어, 쉐어웨어, OEM 소프트웨어는 라이선스 확인하기 어려움 : 소프트웨어와 함께 제공된 책자, 증서, CD 확인  →  설치시 최종사용자계약서 확인

     →   웹사이트 찾아가서 확인  →  설치 폴더에서 확인. (대부분 설치시 최종사용자계약서 제시함)

  - 가정 등 한정된 장소에서 사적인 복제 허용 : 비영리 및 종교단체는 대상이 아님 (음원, 영상은 일부 허용, 소프트웨어는 해당사항 없음)

  - 복구CD, DSP CD, COEM CD, OEM CD 등은 하드웨어와 일체로 판매되므로 분리되어 판매되지 않음

  - 인터넷 쇼핑몰에서 소프트웨어 CD만은 별도 구입시 양도가 가능한지 확인 필요

  - 박스 패키지 : 박스 자체는 보관 필요 없음, 정품CD와 라이선스 필요

  - OEM RW 라이선스를 회사에 설치 : 라이선스 정책 확인 필요 (nero 대상 제한 없음, powerDVD는 상업적 용도 사용불가)

  - 윈도우 7 홈에디션은 회사에서 사용 가능 : MS 라이선스 정책으로 타제품에는 적용 안됨

SW 설치 및 사용 (7)

  - 잠시 설치하는 것도 저작권 위반 : 판례) 사용기간에 상관없이 정품 소매가격을 전액 배상해야한다고 판결

  - 불법복제 설치하지 않고 가지고만 있어도 저작권 침해 : 복제권 침해 (혹시 웹하드 등에 올렸다면 전송권도 침해)

  - 정품 소프트웨어 구매했는데, 크랙이용 : 법적으로 문제될 수 있음

  - 멸실의 경우 복제해 둔 것도 사용 가능하나 멸실을 증명하는 것은 사용자가 해야함

  - 단종된 제품도 저작권 보호됨 (70년)

  - 자신이 설치하지 않았다고 하더라도 

  - 고객이 정품 가진 줄 알고 깔아주는 것 : 알고했으면 처벌될 수 있음, 자기 것으로 설치해 줬다면 복제권 침해임

SW 저작물의 보호 (6)

  - 저작권과 특허권 보호 차이 : 저작권은 아이디어를 보호하지 않음, 특허권은 아이디어에 대해 독점배타적인 권리 보호

  - 특허법상 발명 : 신규성,  진보성(통상의 지식 가진 자가 용이하게 발명할 수 없어야 함),  산업상 이용 가능성

  - 저작권은 별도의 등록 절차 없음, 한국저작권위원회에 등록하면 법적 지위에 몇 가지 이점 부여

  - 소프트웨어가 구현된 장치 혹은 구현되는 방법은 특허로 등록되어 보호됨

  - 국제조약에 가입되어 있는 다른 나라에서도 우리 국민이 창작한 소프트웨어가 저작권 보호됨 (베른협약과 세계지적재산권기구 저작권조약 : WCT, WIPO Copyright Treaty)

  - 저작권법이 적용되는 기계의 범위에는 컴퓨터만 국한하는 것이 아님

  - 원시코드 및 목적코드 모두 저작권 보호 대상

  - 등록시 이점 : 저작자 및 창작연월일 추정, 창작사실의 명확화, 침해사실의 과실추정, 제3자에 대한 대항력 등의 이점
