소프트웨어 개론

   소프트웨어 개요

      컴퓨터 프로그램보다 넓은 개념인 소프트웨어 : 프로그램 매뉴얼과 같은 보조적인 문서 형태의 산출물을 포함

      소프트웨어의 특징 : 비가시성,  복잡성,  변경성(필요에 따라 지속적인 수정 가능),  장기간(비마모성),  복제 가능

      컴파일 / 인터프리터(코드 노출 우려)

      시스템 소프트웨어 (컴퓨터와 상호작용)  vs  응용 소프트웨어 (사용자와 상호작용)

      소프트웨어 = 프로그램(프로그램저작물) + 관련자료(어문저작물)

 

   소프트웨어 분류

      산업/품목에 따른 소프트웨어 분류

         패키지 소프트웨어와 주문 제작 소프트웨어

            PMBOK 구매 관리 프로세스 :

                - 조달계획수립

                - 공급자유치계획수립 : 제안요청서, 입찰안내서, 견적 가격 제출 요청

                - 공급자유치 : 입찰설명회

                - 공급자선정 : 선정방법 (screening:최소기준미달제외, weighting:가중치평가,,,)

                - 계약관리

                - 계약종결/사후관리

         소프트웨어 산업 분류 체계 (통계분류포털.표준산업분류)

         소프트웨어 품목 분류 체계 (ICT 통계 분류 체계)

         서버 소프트웨어 : 운영체재,  DBMS,  WEB운영,  보안솔루션,  클라우드컴퓨팅

      사용 범위에 따른 소프트웨어 분류 : 상업용, 쉐어웨어, 프리웨어, 공개소프트웨어

      기업용 소프트웨어 분류

         ERP : 전사적 자원관리,   BPR(비즈니스 프로세스 재설계)

         SCM : SCP(수요 예측을 바탕으로 최적화된 생산 계획),  SCE(물류 중심으로 현장 관리)

         CRM : 시장점유율 중심이 아니라 개별적인 고객 중심 (이메일, 텔레마케팅 등을 통한 개별 마케팅)

         SEM(전략적경영관리) : DW,  VBM(회계매출 중심이 아니라 장기적인 가치 중심), ABM(활동원가관리),  BSC(조직 지표관리)

         BPM : 표준 모델링 도구 활용, 비즈니스 프로세스 가시화, 워크플로우 활용

소프트웨어 공학

   소프트웨어 생명주기(SDLC) 모델

      폭포수모델 : 각 단계 종료되면 이를 검증한 후 다음 단계 진행, 요구사항분석-설계-구현-시험-유지보수

          관리가 용이, 이해하기 쉬움  vs  초기 고객요구사항 구체적으로 정의하기 어려움,  초기단계 강조하면 코딩/테스트 지연 발생

      프로토타이핑모델 : 주요기능 조기식별, 시제품 개발후 사용자 평가하여 재정립

          요구사항 분석 효율적임, 사용자와 의사소통 원활,

          품질 향상 가능  vs  중간단계산출물 문서화 안되는 경향,  중간단계 폐기시 개발공수 낭비

      나선형모델 : 프로토타입을 폐기하지 않고 지속적으로 발전시킴, 계획및정의-위험분석-개발-고객의평가

         주기적인 위험평가로 위험 감소  vs  프로젝트관리복잡

 

   소프트웨어 개발 방법론

      개발자 임의로 코딩

      구조적개발방법론 : 대규모 개발에 한계

         구조적분석(자료흐름도, 개체관계도, 상태전이도, 소단위명세서, 자료사전) - 구조적설계 - 구조적프로그래밍(순차,조건,반복)

      정보공학개발방법론 : 도입효과 확인까지 장기간 소요, 소규모 개발 부적합, 공학적 기법 도입

         정보전략계획(ISP) - 업무영역분석 - 업무시스템설계 - 업무시스템구축

      객체지향개발방법론 : 전문가집단 부족, 개발자 학습 기간 필요

          프로세스+데이터 -> 객체로 분석, 소프트웨어 생산성과 유지보수 어려움 극복하고자 함

          추상화,  캡슐화,  상속성,  다형성

      컴포넌트기반개발방법론 : 재사용 가능한 컴포넌트 부족, 다른 플랫폼에서는 호환 안됨

           -> XML 기반 웹서비스,  컴포넌트 수준 재사용 넘기 위해 프레임워크 기반 방법론 등장

      프로젝트관리방법론 : PMBOK

          통합관리,  범위관리,  시간관리,  비용관리,  품질관리,  인적자원관리,  의사소통관리,  위험관리,  구매관리,  이해관계자관리

 

소프트웨어 산업 역사와 최신 동향

   소프트웨어 산업 역사   

      소프트웨어 개발 용역 시대 (1950년대~)

         1950년대 이전 : 천공카드를 이용한 프로그래밍

         1950년대 이후 : 프로그래밍 언어와 컴파일러를 활용한 프로그래밍

            (1957 포트란,  1958 LISP, ALGOL,  1960 COBOL:시스템독립적인소프트웨어)

         1960년대 이후 : 멀티프로그래밍 및 시분할 방식 등장 (배치방식이 시간 많이 걸리는 것 해결)

         용역형태 개발 비즈니스 : 개발전문업체, 하드웨어 업체가 주고객, SW개발에 드는 인력과 기간등에 의해 가치 산정

         초기에는 하드웨어 번들  →  상품 또는 지적자산으로 가치 부여 받음

      패키지 소프트웨어 시대 (1970년대~)

         SAP(1972),  컴퓨웨어(1973),  마이크로소프트(1975), CA(1976), SAS(1976), 오라클(1977)

         C언어와 Unix 그리고 PC  /  MS-DOS, Windows 95  /  Linux 1991 리누스 토발즈

         소프트웨어 위기 : 하드웨어의 발전 속도를 따라가지 못함

            요구사항이 커지고 높아짐  /  규모가 커짐에 따라 분석개발 일정 지연, 비용 증가  /  유지보수 복잡해지고 비용 증가

      소프트웨어 서비스 시대 (1990년대~)

         인터넷혁명,  데이터센터,  XML기반 SOA(웹서비스),  소프트웨어 통합개발환경(ALM), ASP, SaaS, 클라우드

 

   소프트웨어 최신 동향

      IT신기술

         빅데이터(데이터 수집, 저장, 분석, 보고),  IoT(초연결),  핀테크,  시맨틱웹,  딥러닝

      소프트웨어 패러다임 변화

         IT 컨버전스화 : 인프라와 어플리케이션 통합 : On-demand, 서비스로의 소프트웨어(SaaS)

         구매차 중심의 소프트웨어 시장 : 소비자(기업고객) 비용절감 -> 대형 벤더 유리

         하드웨어를 대체하는 소프트웨어 : 임베디드

 

   정보통신산업진흥원(NIPA) : 정보 통신 연구진흥원, 한국소프트웨어진흥원, 한국전자거래진흥원이 통합 (2009.8)

   소프트웨어정책연구소(SPRi)

   소프트웨어산업 연간 보고서(2011-2018)  -  소프트웨어산업백서 (2007 - 2010)
