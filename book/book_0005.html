Pragmatic : solving problems in a practical and sensible way rather than by having fixed ideas or theories

1. 실용주의 철학

- 철학

- 프로젝트에는 문제가 당연히 있고, 해결책 위주로 문제 바라봐야

- 문제를 보이게 하고 소통될 수 있게 해야 (돌멩이스프)

- 완벽한 것을 바라지 말고 적당한 프로그래밍으로 (적절한 품질)

- 기술 급변하고 있음  현재 흐름 놓치지 않게 공부

- 나의 부족점 : 요증 흐름을 놓치지 않기, 고인 지식이 되지 않게 소통이 중요하다

4. 실용주의 편집증

- 완벽한 소프트웨어는 없다

- 계약에 의한 설계 : SHY, DBC가 뭔지?

- 죽은 프로그램은 거짓말을 하지 않는다 : 문제 있다고 판다되면 멈춰라

- 단정적 프로그래밍 : ASSERT

- 언제 예외를 사용할까 : TRY … CATCH … 최소화 필요

- 리소스 사용의 균형 : 시작한 쪽에서 책임지고 끝내라
