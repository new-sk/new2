    CSS 선택자

    전체 선택자 (*)
    * { color: red; } <!-- 모든 내용을 빨간색으로 만듦 -->

    유형 선택자 (태그명칭) : 일반적으로 사용하는 것
    h1 { color: red; } <!-- h1만 빨간 글자로 -->

    아이디(#) vs 클래스(.) : 아이디는 고유한 값을 가짐 vs 하나의 클래스에 여러 태그 지정 가능, 하나의 태그에 여러 클래스 지정 가능
    .my_class1 { color: red; }
    <p class="my_class1 my_class2">여러 태그를 하나의 클래스로 묶을 수 있고, 태그 하나에 여러 클래스 적용 가능</p>

    복합 선택자
    조상자손선택자 (조상태그명칭 자손태그명칭)
    div p { color: red } <!-- div 태그의 자손인 p 태그의 색깔을 빨간색으로 지정 -->

    08. CSS 배치 속성과 반응형 웹

    block vs inline-level element

    h1, span { border : 3px solid red; } <!-- h1, span 태그에서 경계선을 지정함 -->

    h1 : block (block은 줄바꿈 일어남, inline은 요소 크기만큼 공간 차지하여 가로 줄 차면 줄바꿈 이루어짐)

    span : inline (속성만 잠시 바꿀 때)

    display 속성을 변경하여 강제로 변경할 수도 있음 { display: inline; }

    박스 모델

    모든 HTML 요소에서 적용됨

    요소(element), 패딩(padding), 테두리(border), 마진(margin) : 패딩과 마진은 투명색

    브라우저 검사 기능을 통하여 속성 확인 가능함 (F12, 도구더보기-개발자도구)

    반응형 페이지 : 미디어쿼리 사용 : 접속하는 기기 해상도에 따라서 HTML과 CSS가 바뀌는 웹을 반응형 웹이라고 함

    @media (max-width: 1280px) { body { background-color: pink } } <!-- 너비가 1280px 이하인 경우 배경색이 핑크 -->

    @media (max-width: 600px) { body { background-color: pink } } <!-- 위에서부터 적용, 위는 600~1280, 여기는 600이하 -->

    13. 깃/깃허브 배우기

    버전관리시스템, 깃의 클라우드 서비스 버전 깃허브, 2018년 MS에 인수

    SaaS 형태로 서비스 제공 : 깃은 코드 버전 관리 프로그램 : 깃허브는 깃 소프트웨어 기반으로 구축된 클라우드 서비스

    저장소(repository) : 저장하는 공간

    브랜치(branch) : 버전별 저장소 (메인 브랜치 / 서브 브랜치) (용어변경 '20.10 마스터->메인)

    클론(clone) : 깃허브 저장소에 있는 파일들을 로컬 컴퓨터로 복사본 만드는 것

    애드(add) - 커밋(commit) - 푸시(push) : 저장소로 변경된 파일을 보내는 방식

    풀(pull) : 클론은 처음, 그 이후에는 풀

    1. 저장소 만들기

    2. 클론하기 (처음 전체 소스를 받을 때)

    3. 파일 추가하기

    4. 커밋 및 푸시하기

    5. 풀하기 (클론한 이후 변경된 내용만 받을 때)
